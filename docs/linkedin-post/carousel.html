Decisões de Engenharia 04 / 08
Por que ML
não supervisionado?
Sem rótulos históricos de fraude confirmada, a abordagem supervisionada não é viável. Estas foram as três decisões centrais de engenharia:
Isolation Forest para detecção de anomalias
Escolhi IF porque isola instâncias raras com menos splits na árvore — eficiente quando fraude é minoria absoluta dos dados e sem labels disponíveis.
K-Means + DBSCAN para segmentação de perfis
K-Means define clusters com número fixo de grupos de risco (5 clusters); DBSCAN encontra agrupamentos por densidade sem número pré-definido — abordagens complementares.
Risk Score Engine multi-fator (escala 0–100)
Missing rate (25%) · anomaly score (25%) · padrão histórico (20%) · cluster risk (15%) · frequência (15%). Pesos calibrados para operações de entrega e-commerce.
